# CONSENSUS_qingluan-marketing-site

## 明确需求描述

实现一个基于 Next.js + TypeScript + Tailwind CSS 的英文品牌营销网站，用于展示青鸾书院（Qingluan Academy）的三项核心咨询服务：Bazi、Qimen Dunjia、Feng Shui。网站需以现代高端、文化克制、可信清晰的风格呈现，并通过明确 CTA 引导用户进入咨询或预约流程。

## 验收标准

- 存在以下页面并可正常访问：
  - `/`
  - `/about`
  - `/services`
  - `/services/bazi`
  - `/services/qimen-dunjia`
  - `/services/feng-shui`
  - `/faq`
  - `/contact`
  - `/book`
- 首页包含：
  - Hero
  - 价值主张
  - 服务概览
  - How it works
  - 信任理由
  - 适合人群
  - CTA
  - 免责声明页脚
- 服务总览页展示 3 个服务卡片并链接到详情页
- 每个服务详情页包含标题、摘要、适合人群、覆盖内容、流程、交付物、价格占位、CTA
- 提供 FAQ、联系表单 UI、预约占位页
- 导航清晰，Book 按钮有视觉强调
- 页面在移动端与桌面端均可良好展示
- 包含页面级元数据与基础 SEO
- 风格不落入通用科技模板或廉价玄学模板

## 技术实现方案

- 使用 Next.js App Router
- 使用 TypeScript 严格类型
- 使用 Tailwind CSS 实现设计系统
- 使用本地 mock 数据驱动页面文案和服务信息
- 通过复用组件保持页面一致性
- 以静态渲染优先，保留后续扩展空间

## 技术约束与集成方案

- 本轮不接入后端服务
- 本轮不接入第三方预约 API，仅保留占位入口
- 表单只做前端展示和交互准备
- 元数据通过 Next.js 页面元数据能力实现
- 所有链接必须可达，不允许断链

## 任务边界限制

- 不实现真实业务流转、邮件发送、数据库持久化
- 不实现 CMS
- 不实现多语言切换逻辑
- 不实现账号系统

## 已确认假设

- 当前项目可以从零开始搭建
- 用户接受先交付高质量首版前端站点
- `AGENTS.md` 即为当前最高优先级项目规范

## 当前增量任务共识

- 联系表单将采用项目内真实后端处理，而不是跳转第三方表单页
- 当前邮件服务切换为 Brevo 事务邮件 API
- 实际收件地址、发件地址和 Brevo API Key 通过 `.env` 提供
- 若服务端缺少必要环境变量，接口需明确返回“未配置完成”，而不是静默失败
