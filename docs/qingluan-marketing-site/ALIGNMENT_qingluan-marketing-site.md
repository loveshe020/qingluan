# ALIGNMENT_qingluan-marketing-site

## 原始需求

基于 `AGENTS.md` 为青鸾书院（Qingluan Academy）构建一个面向国际英文用户的高品质营销网站。网站需体现传统文化根基与现代产品思维，避免廉价占卜风格，突出 Bazi、Qimen Dunjia、Feng Shui 三类服务，并具备清晰导航、转化路径与后续扩展能力。

## 项目现状

- 当前仓库仅包含需求文档 `AGENTS.md`
- 暂无现成代码、设计系统、测试体系或构建脚本
- 属于新项目启动场景

## 技术与架构理解

- 推荐技术栈为 Next.js + TypeScript + Tailwind CSS
- 由于当前无需后端，首版适合做静态营销网站
- 页面需要完整可访问，不应存在空链接或未处理占位
- 组件需要可复用，便于后续扩展预约系统、CMS、双语内容

## 业务与品牌理解

- 品牌名：青鸾书院 / Qingluan Academy
- 服务领域：Bazi、Qimen Dunjia、Feng Shui
- 品牌关键词：冷静、优雅、文化根基、现代、可信、专业
- 目标用户：英语用户为主，包括海外华人、创意人士、专业人士、对东方哲学或人生方向感兴趣的人群
- 核心转化目标：让首次访问者快速理解品牌、服务对象、服务内容和预约方式

## 范围边界

本轮包含：
- 前端营销站点搭建
- 主页和 8 个核心路由
- 组件化 UI
- 基础 SEO
- 占位式预约和表单 UI
- 合规免责声明

本轮不包含：
- 实际预约平台集成
- 后端 API
- CMS
- 多语言切换
- 真实支付、认证或用户系统

## 当前增量任务

将 Contact 页的咨询表单从展示型占位改为真实可提交表单。范围包括：

- 前端表单状态与提交反馈
- 服务端 API 接口
- 服务端输入校验
- 基于 Brevo API 的真实邮件投递
- `.env` 配置模板与文档说明

本次增量不包含：

- 数据库存储
- 管理后台
- 自动回复模板系统
- 垃圾邮件风控平台接入
- CRM 集成

## 关键约束

- 所有公开文案使用自然、专业的英文
- 风格需高端、克制、东方文化感，但避免陈词滥调
- 不可使用夸张神秘主义表达
- 需要移动端优先、响应式、可访问性良好
- 代码需模块化，后续易扩展

## 歧义识别与决策

### 已自主决策

- 由于仓库无现成技术栈，首版将直接初始化 Next.js App Router 项目。
- 因用户要求“参照文档进行工作”，默认目标是直接落地首版站点，而不是仅写文档。
- 由于暂无后端，本轮联系表单与预约页将采用高保真占位交互，不提交真实数据。
- 由于未给出品牌标准字或真实图片资源，视觉上采用代码实现的纹理、渐变、几何与装饰元素。

### 待澄清项

当前没有必须中断的问题。若后续出现以下情况需中断确认：
- 用户提供真实品牌中文名编码与当前文档显示乱码不一致
- 用户指定要接入特定预约系统、表单系统或部署平台
- 用户要求保留既有 UI/品牌资产但仓库尚未提供
